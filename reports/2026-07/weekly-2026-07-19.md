## 本周 Top5

### 1. How we contain Claude across products

过去一年，Anthropic 对 AI 智能体（agent）的风险认知发生了根本转变：从完全拒绝授予可能破坏内部服务的权限，到如今将其视为日常开发工具。这篇文章系统阐述了他们如何通过“遏制”（containment）而非单纯“监督”来控制 Claude 智能体的破坏半径（blast radius，即单次故障可能造成的最大损失），使其能在生产环境中安全地执行高风险操作。

> 评分 9/10 · Anthropic 官方工程博客分享跨产品 agent 安全围栏的实战经验，直接解决 agent 爆炸半径这一生产核心问题，方法可复现
> 来源：<https://www.anthropic.com/engineering/how-we-contain-claude> · 发布 2026-07-19

### 2. LongStraw: Long-Context RL Beyond 2M Tokens under a Fixed GPU Budget

随着大模型推理上下文长度向百万 token 级别迈进，强化学习后训练（RL post-training）却普遍停留在 256K token 以下，只能依赖部署时的长度泛化能力来弥补。LongStraw 正是为了弥合这一鸿沟而设计的架构感知执行栈，它能在固定 GPU 预算下完成百万 token 级别的 RL 后训练，特别针对 AI 智能体这类需要累积长轨迹观测、工具输出和历史决策的场景。

> 评分 9/10 · 长上下文 RL 后训练突破 2M tokens，upvotes 极高，直接解决 agent 长轨迹训练痛点，方法可复现
> 来源：<https://huggingface.co/papers/2607.14952> · 发布 2026-07-16

Gemini 3.5 Flash 模型新增了“计算机使用”能力，使其能够直接操控图形界面、执行多步操作。这项变化让模型从纯文本对话扩展到可自主完成网页任务等实际应用，标志着AI代理能力的重要进步。它值得关注，因为这意味着大模型开始具备像人一样操作软件的能力，为自动化工作流和智能助手开辟了新路径。

## 本周趋势

本周最显著的走向是智能体从实验性 Demo 向生产级系统加速演进，安全与工程化成为核心议题。Anthropic 公开了其分层遏制体系，用环境层硬隔离替代单纯依赖审批，将破坏半径压缩到最小，并量化了审批疲劳（93% 批准率）和沙箱收益（84% 提示减少），标志着智能体安全从“人在回路”转向“环境硬边界”。与此同时，大量工程条目围绕智能体评估、记忆、工具使用和可观测性涌现，LangChain 等平台密集发布评估基准、动态子智能体、循环工程等实践，说明行业正从“能不能做”转向“如何稳定可靠地做”。另一条线索是模型能力向长上下文和动手操作延伸：LongStraw 在固定 GPU 预算下实现 200 万 token 级别的 RL 训练，为智能体长轨迹优化提供了可行路径；Gemini 3.5 Flash 新增计算机使用能力，GPT-5.6 Sol 和 Seed2.1 则分别在编程、科学推理和端到端代码交付上深度强化，模型正从“对话者”变为能直接操控环境、完成复杂任务的执行者。

## 面试视角

第一，谈智能体安全架构时，可引用 Anthropic 的“破坏半径”和分层防御理念，说明在生产环境中应优先在环境层设置硬边界（如凭证不入沙箱、网络默认禁止），而非仅依赖模型层概率防御或审批机制，这直接关系到架构选型与风险控制。第二，面对长上下文智能体训练的资源瓶颈，可借 LongStraw 的“一次评估、多次回放”策略，展示“计算换空间”的工程权衡思维，并结合 GRPO 等低显存 RL 算法，讨论如何在有限 GPU 预算下设计训练方案，体现成本与可行性的平衡。第三，在模型选型与评估上，可结合 GPT-5.6 Sol 的垂直能力强化和 Seed2.1 的端到端代码交付稳定性，提出面试中应关注模型在具体业务场景（如自动化办公、代码生成）中的可靠交付能力，而非仅看通用榜单，并强调需建立面向智能体多步执行的评估体系，如 Workspace Bench 等，以支撑实际应用决策。
