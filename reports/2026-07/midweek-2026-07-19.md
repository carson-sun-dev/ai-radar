### ⭐ 深读 | Quantifying infrastructure noise in agentic coding evals Feb 05, 2026

这是Anthropic工程团队发布的一项关于智能体编程评估（agentic coding evals）中基础设施噪声的量化研究。他们发现，在SWE-bench和Terminal-Bench这类流行的编程能力基准测试中，仅凭运行环境的资源配置差异，就能造成高达6个百分点的分数波动，这一差距已超过许多排行榜上模型之间的领先幅度。

此前的痛点在于，业界习惯将基准测试分数视为模型能力的精确度量，并据此做出部署决策。但传统的静态基准测试只评估模型输出，而智能体编程评估不同：模型在一个完整的运行时环境中编写程序、运行测试、安装依赖并多轮迭代。这里的运行时不再是“被动容器”，而是问题解决过程的组成部分。不同资源预算和时间限制下的智能体，实际上参加的不是同一场考试。虽然Terminal-Bench 2.0等基准已开始按任务推荐CPU和内存规格，但“推荐”不等于“一致执行”，执行方式本身就会改变评估的实际测量内容。

研究团队在Google Kubernetes Engine集群上运行Terminal-Bench 2.0时，发现了关键的技术细节。容器运行时通过两个独立参数来执行资源限制：保证分配量（预先预留的资源）和硬性终止阈值（容器被杀死时的上限）。当两者设为相同值时，容器没有任何应对瞬时资源峰值的余量，一次短暂的内存波动就可能导致OOM（内存溢出）杀死容器。研究设置了从严格1倍执行到完全无上限的六种资源配置，发现成功率随资源余量增加而提升。在约3倍推荐规格之前，额外资源主要解决基础设施可靠性问题，将基础设施错误率从5.8%降至2.1%，但成功率波动仍在噪声范围内。超过3倍后，额外资源开始主动帮助智能体解决此前无法解决的问题——智能体可以拉取大型依赖、生成重量级子进程、运行内存密集型测试套件，成功率提升幅度超过错误率下降幅度。在SWE-bench上的交叉验证也发现了相同趋势，尽管幅度较小（1.54个百分点），因为其任务资源密集度较低。

对AI应用岗而言，这项研究揭示了生产环境中模型评估的一个根本性挑战：基准分数正在越来越多地影响技术决策，但小分差所携带的不确定性远超数字精度所暗示的。面试中可以谈三点：第一，智能体评估本质上是端到端系统测试，任何组件都可能成为混淆变量，包括集群健康度、硬件规格、并发级别甚至出口带宽，团队甚至观察到通过率随一天中的时段波动；第二，资源限制会改变评估的实际测量内容——严格限制奖励编写精简高效代码的策略，宽松限制奖励能充分利用资源的“暴力”策略，两者都是合理的能力维度，但混为一谈会让分数的现实泛化性难以解释；第三，工程实践上应把资源配置作为一等实验变量，像提示词格式或采样温度一样被记录和控制，建议基准测试同时公布保证分配量和硬性终止阈值两个参数，并校准两者之间的区间，使分数波动落在噪声范围内。一个可引用的具体数据是：在Terminal-Bench上，低于3个百分点的排行榜差异值得怀疑，直到评估配置被记录并匹配。

### ⭐ 深读 | LongStraw: Long-Context RL Beyond 2M Tokens under a Fixed GPU Budget

随着大语言模型推理上下文窗口迅速向百万 token 级别迈进，强化学习后训练（RL post-training）的上下文长度却普遍停留在 256K token 甚至更低，只能依赖部署时的长度泛化能力来弥补。LongStraw 正是针对这一断层而设计的执行栈，目标是在固定 GPU 预算下，让 RL 后训练能够处理百万 token 级别的长上下文。

此前的核心痛点在于，长上下文训练的内存开销极大，尤其是 GRPO（Group Relative Policy Optimization，一种基于组内相对比较的策略优化方法）这类需要同时为多个响应分支维护计算图的方法，很容易超出 GPU 显存上限。对于 AI 智能体而言，长轨迹中积累的观察、工具输出和历史决策无法被完整纳入训练，导致模型在真实长程任务中表现受限。

LongStraw 的技术要点在于“架构感知”的显存优化策略。它首先对共享的 prompt 部分进行一次无自动微分的评估，只保留后续 token 生成所必需的模型状态，而丢弃完整的计算图。随后，它将多个响应分支逐个重放，每次只维护一个分支的活跃训练图，用额外的重放时间换取显存空间的大幅缩减。这一设计在 Qwen3.6-27B（混合循环与全注意力架构）和 GLM-5.2（压缩注意力混合专家架构）上得到验证：在 8 张 H20 GPU 上，处理 2.1M token 的 prompt 并完成分组评分与响应反向传播时，将组大小从 2 增加到 8 仅增加 0.21 GB 峰值显存，压力测试更达到 4.46M token；在 32 张 H20 上，GLM-5.2 的全 78 层端到端执行路径也得到验证。

对 AI 应用岗而言，这条工作的面试价值在于它直面了“训练-推理上下文断层”这一生产级问题。面试中可以讨论：长上下文 RL 后训练对智能体、长文档理解等场景的必要性；GRPO 等策略优化方法在长上下文下的显存瓶颈；以及 LongStraw 所展示的“重放换显存”思路如何在不增加硬件成本的前提下扩展训练规模。需要注意的是，论文目前验证的是执行容量而非完整训练正确性，因为捕获的 prompt 状态是分离的，部分分布式前向和梯度组合路径尚未完全打通，这恰好可以成为面试中探讨工程落地挑战的切入点。

### 中读 | Bridge Evidence: Static Retrieval Utility Does Not Predict Causal Utility in Multi-Step Agentic Search

Bridge Evidence 这篇论文揭示了一个关键发现：在静态检索评估中看似无用的文档，在多步智能体搜索中却可能承担关键的因果作用。研究通过对比静态检索效用与反事实轨迹效用，发现两者近乎统计独立，并识别出约三分之一的文档属于“桥接文档”——它们对静态读者无价值，却是智能体完成后续推理和查询所必需的。这一发现挑战了当前检索系统仅依赖静态有用性进行训练和评估的范式，强调了需要为智能体搜索设计能捕捉文档动态、因果效用的新评估方法。
